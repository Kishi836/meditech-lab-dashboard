Welcome to Meditech Lab

A hands-on dashboard for the four core pillars of medical informatics, running over Bangalore City Hospital's real PostgreSQL database. Pick a tab on the left to explore — here's what each one does and what to try first.

Capture data → Store & query it → Extract from notes → Code to standards → Exchange between systems

Patients

The hospital's electronic health record. Search the registry and open a patient to see demographics, past visits, lab-result trend charts, conditions and medications — read live from Postgres.

Concept Storing & querying structured patient data.

▸ Try: search Menon, open Rajan Menon, then the Labs tab to watch his HbA1c climb over the years.

Intake

Real data

Add real, permanent data the way a hospital does: Register Patient emits an ADT^A04 admission message and Record Result emits an ORU^R01 lab result — each one built live as you type, then sent through the full six-stage pipeline before the row lands in Postgres and shows up under Patients.

Concept Data capture as document flow — in real hospitals, new records enter as HL7 messages, not raw database writes.

▸ Try: register a patient and watch the packet travel the pipeline, then record an HbA1c of 9.5 % for them — it's flagged critical and shows up in the Pipeline tab's critical-values panel.

Extractor

Paste a free-text clinical note — the messy way doctors actually write — and it pulls out the structured bits (vital signs, lab values, diagnosis codes), highlights them in the text, and lists them in a table.

Concept Clinical information extraction — turning unstructured text into data.

▸ Try: load the sample note, hit Extract, and see the vitals and codes light up.

Coder

Look up ICD-10 diagnosis codes and SNOMED clinical terms, browse how a concept fits in its hierarchy, and cross-map between the two coding systems.

Concept Standardized medical coding & terminology.

▸ Try: search diabetes, open a concept, and see its ICD-10 cross-map (e.g. E11.9).

Pipeline

Sandbox

A demo sandbox for interoperability: build an HL7 v2.5 message from a patient, then watch it flow through a hospital integration stack — routing → database → search index → dashboards → archive — and drive the live feed + analytics. Sends here are tagged practice rows.

Concept Interoperability — the HL7 messaging that keeps hospital systems in sync.

▸ Try: pick PT-001 + ORU^R01, hit Send, watch the six stages animate and the feed/charts update. Reset wipes these demo sends — to enter records that stay, use Intake.

How it's built

  • Backend: Python + Flask, one blueprint per module, talking to PostgreSQL through parameterized queries (no raw string SQL).
  • Frontend: a server-rendered shell + vanilla JavaScript, charts by Chart.js, animated SVG pipeline diagram, in this "Clinical Light" theme.
  • Pipeline realness: build and the Postgres insert are real; Elasticsearch is real when enabled; NiFi, Kibana and MinIO are simulated in-app (MinIO writes the raw HL7 to a local archive file).
